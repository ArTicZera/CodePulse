Linguagem Assembly - Registradores

O que sao registradores?

Em assembly, registradores sao areas de armazenamento de dados temporarios localizados dentro da CPU (Unidade Central de Processamento). Isso garante uma alta velocidade pois ele nao utiliza a memoria como armazenamento.

Tipos de registradores

Ha diversos registradores na CPU para diversos tipos de operacoes, veja abaixo:

  • Registradores Gerais: Usado para armazenar dados temporarios e realizar operacoes aritmeticas e logicas.
  • Registradores de Segmento: Usado para controlar o acesso a diferentes partes da memoria.
  • Registradores de Indice: Usados em operacoes de acesso a arrays e estruturas de dados.
  • Registradores de Ponto Flutuante: Usados para operacoes com numeros de ponto flutuante (decimais).
  • Registradores de Status: Armazenam informacoes sobre o estado atual da CPU, como flags de condicao apos uma operacao.

Registradores Gerais

Em uma arquitetura x86, temos esses registradores gerais:

EAX, EBX, ECX e EDX sao registradores de 32 bits

AX, BX, CX e DX sao registradores de 16 bits

AH/AL, BH/BL, CH/CL e DH/DL sao registradores de 8 bits


Repare que os registradores de 16 bits estao divididos em duas partes. o AX, por exemplo, esta dividido em AH e AL.

H vem de High Byte e L vem de Low Byte

Essa divisao facilita o acesso a dados de tamanho menor, operacoes com bytes individuais, economia de espaco e compatibilidade.

Registradores de Segmento

Sao registradores para controlar o acesso a diferentes segmentos de memoria, sendo importantes para o enderecamento de memoria e na gestao de acesso a diferentes partes. Sao eles:

  • CS (Code Segment): Aponta para o segmento de codigo, que contem as instrucoes de programa em execucao
  • DS (Data Segment): Aponta para o segmento de dados, que contem dados estaticos e variaveis globais
  • ES (Extra Segment): Eh usado para acesso adicional a dados, geralmente usado em instrucoes que lidam com operacoes de strings ou matrizes.
  • SS (Stack Segment): Aponta para o segmento de pilha, mas nao se preocupe, veremos sobre isso em breve!
  • FS e GS: Em algumas arquiteturas x86, existem tambem os registradores FS e GS, que fornecem segmentos adicionais para acesso de dados.

Registradores de Segmento

Sao registradores para controlar o acesso a diferentes segmentos de memoria, sendo importantes para o enderecamento de memoria e na gestao de acesso a diferentes partes. Sao eles:

  • SI (Source Index): Eh usado como indice para percorrer uma string ou uma array de dados.
  • DI (Destination Index): Eh usado em operacoes semelhantes ao SI, mas eh geralmente usado para armazenar o destino de uma operacao, como copia de uma string para outra.
